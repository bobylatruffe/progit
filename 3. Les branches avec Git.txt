Comment Git stocke ses données ?
	Il les stockes en instantanés (snapshots)

Qu'est-ce qu'un commit d'un point vue "technique" ?
	C'est un objet qui contient un pointeur vers l'instantané en question.

Qu'est-ce qu'une branche ?
	Une branche c'est un pointeur léger et déplaçable vers un commits.

Qu'est-ce que la branch master ?
	C'est le nom de la branche principale quand on fait un git init.
	C'est un pointeur qui se déplace automatiquement sur le dernier commit.

Qu'est-ce qu'il se passe si je fais un git branch testing ?
	Cela crée une nouvelle branche, càd un pointeur sur le commit en courant.
	Mais ne déplace pas le pointeur spécial HEAD sur celle-ci.

Qu'est-ce que le pointeur HEAD ?
	C'est un pointeur qui indique sur quel branche nous nous trouvons en local.

Comment basucler sur une autre branche ?
	git checkout nom_branche

Comment afficher toutes les commits de toutes les branches avec git log ?
	git log --all

Comment créer une branche et directement se mettre dessus ?
	git checkout -b <nouvelle-branche>
	ou depuis 2.23
	git switch -c nouvelle-branche

Si je dispose d'une B issue d'une branche A, comment fusionner la branche B dans A ?
	git checkout A
	git merge B

Comment afficher la liste des branches courantes ?
	git branch

Qu'indique le caractère * dans la cmd git branch ?
	Elle indique la branche courante (càd le pointeur HEAD).

Comment afficher les derniers commit de chaque branche ?
	git branch -v

Qu'est-ce que ça veut dire qu'une branche est extraite ?
	Que le pointeur HEAD est dessus.

Qu'est-ce qu'il se passe si j'ai deux branches et que je modifie la même partie d'un fichier, et que je merge ?
	Il y'a un conflict, qu'il faut résoudre.

Que permet l'option --merged de la cmd git branch ?
	Elle permet de voir les branches qui ont été fusionné avec la branche courant.
	Nous pouvons supp les branches qui ne possède pas de caractère *, car elles ont été mergé.

Comment renommer une branche localement ?
	git branch --move ancien_nom nouveau_nom

Comment renommer une branche sur un dépot distant ?
	git push --set-upstream remote_name nouveau_nom_branch
	il faut ensuite supp l'ancienne avec 
		git push remote_name --delete ancien_nom_branch
